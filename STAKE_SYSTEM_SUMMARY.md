# Система изолированных валют (как в Stake)

## ✅ Что уже реализовано

### 1. База данных

**Таблица Balance** уже поддерживает изоляцию валют:
- Уникальность: `@@unique([userId, tokenId, type])`
- Каждый пользователь имеет отдельный баланс для каждой валюты (tokenId)
- Два типа балансов: MAIN и BONUS для каждой валюты

**Таблица UserWallet** (новая):
- Отдельный кошелек для каждой валюты
- Уникальность: `@@unique([userId, tokenId])`
- Хранит адрес кошелька, сеть, приватный ключ (зашифрованный)

### 2. API Endpoints

#### Получение баланса по валюте:
```
GET /api/v1/balance/balance/:tokenId
```
Возвращает MAIN + BONUS баланс для конкретной валюты

#### Получение всех балансов:
```
GET /api/v1/wallet/balance
```
Возвращает все балансы пользователя по всем валютам

#### Получение/создание кошелька:
```
GET /api/v1/wallet/wallet/:tokenId
```
Получает или создает кошелек пользователя для валюты

#### Получение всех кошельков:
```
GET /api/v1/wallet/wallets
```
Возвращает все кошельки пользователя

### 3. Игры

Все игры теперь поддерживают выбор валюты:
- **Plinko**: `tokenId` в query параметрах или body
- **Minesweeper**: `tokenId` в body запроса
- **Crash**: `tokenId` в body запроса

Функции работы с балансами:
- `getUserBalances(userId, tokenId)` - получает баланс для валюты
- `determineBalanceForBet(userId, betAmount, tokenId)` - определяет с какого баланса списывать
- `deductBetFromBalance(userId, betAmount, tokenId)` - списывает ставку
- `creditWinnings(userId, amount, tokenId, balanceType)` - зачисляет выигрыш

### 4. Пополнение

Система пополнения:
- Сохраняет `asset` и `network` в `PendingDeposit`
- При обработке депозита создает баланс для правильного `tokenId`
- Каждая валюта пополняется отдельно

## 🔄 Как это работает (как в Stake)

### 1. Пополнение валюты

1. Пользователь выбирает валюту (например, BTC)
2. Система создает или получает кошелек для BTC (`UserWallet`)
3. Пользователь пополняет баланс BTC
4. Баланс BTC создается/обновляется в таблице `Balance` с `tokenId = BTC`

### 2. Игра валютой

1. Пользователь выбирает валюту для игры (например, ETH)
2. Игра получает баланс ETH через `getUserBalances(userId, tokenId=ETH)`
3. Ставка списывается с баланса ETH
4. Выигрыш зачисляется на баланс ETH
5. **Важно**: BTC и ETH не пересекаются - это разные балансы

### 3. Вывод валюты

1. Пользователь выбирает валюту для вывода (например, USDT)
2. Система проверяет баланс USDT
3. Вывод происходит только в USDT
4. Другие валюты не затрагиваются

## ✅ Преимущества системы

1. **Полная изоляция валют** - каждая валюта имеет свой баланс
2. **Отдельные кошельки** - для каждой валюты свой адрес пополнения
3. **Нет конвертации** - валюты не пересекаются между собой
4. **Гибкость** - можно добавить любую валюту без изменения логики
5. **Прозрачность** - пользователь видит баланс каждой валюты отдельно
