# Инструкция по применению миграции валют

## Проблема

В базе данных было ограничение `UNIQUE` только на поле `symbol`, что не позволяло хранить несколько токенов с одинаковым символом, но разными сетями (например, USDT TRC-20 и USDT ERC-20).

## Решение

Обновлена схема базы данных для поддержки уникальности по комбинации `symbol + network`.

## Шаги для применения миграции

### Вариант 1: Автоматическое применение (рекомендуется)

1. Остановите сервер (если запущен)
2. Запустите скрипт миграции:
   ```bash
   cd api
   node scripts/apply_currency_migration.js
   ```
3. Перезапустите сервер

### Вариант 2: Ручное применение через Prisma

1. Остановите сервер
2. Примените миграцию:
   ```bash
   cd api
   npx prisma migrate deploy
   ```
3. Перезапустите сервер

### Вариант 3: Через SQL напрямую

Если у вас есть доступ к базе данных напрямую, выполните SQL:

```sql
-- Удаляем старое ограничение
ALTER TABLE "CryptoToken" DROP CONSTRAINT IF EXISTS "CryptoToken_symbol_key";

-- Добавляем новое ограничение на symbol + network
ALTER TABLE "CryptoToken" ADD CONSTRAINT "CryptoToken_symbol_network_key" UNIQUE ("symbol", "network");

-- Добавляем индекс для быстрого поиска
CREATE INDEX IF NOT EXISTS "CryptoToken_symbol_idx" ON "CryptoToken"("symbol");
```

## Проверка

После применения миграции перезапустите сервер и проверьте логи. Должно быть:
- ✅ Все валюты синхронизированы без ошибок
- ✅ В базе данных есть все 27 валют (USDT TRC-20, USDT ERC-20, USDT BEP-20 и т.д.)

## Что изменилось

### До миграции:
- ❌ Мог быть только один токен с символом "USDT"
- ❌ Нельзя было иметь USDT TRC-20 и USDT ERC-20 одновременно

### После миграции:
- ✅ Можно иметь несколько токенов с одинаковым символом, но разными сетями
- ✅ USDT TRC-20, USDT ERC-20, USDT BEP-20 и т.д. - все поддерживаются
- ✅ Всего 27 уникальных валют (symbol + network)
